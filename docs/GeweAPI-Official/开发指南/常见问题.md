# 常见问题

<AccordionGroup>
<Accordion title="本服务是否需要安装第三方插件？"  >
    本服务不需要安装第三方插件，使用官方正版微信即可，[微信官方下载地址](https://weixin.qq.com/)。
 </Accordion>
    
    <Accordion title="微信版本是否要求？"  >
   本服务对于微信版本无要求，官方版本正常使用即可。
 </Accordion>
 
    <Accordion title="每天发送内容数量是否有限制？" >
    服务没有限制，但是微信本身有规则限制，例：发送频率是否过快、发送内容是否违规等，建议开发者做好频率控制，消息发送类必须走队列消费，做好消息间隔，勿并发。
    </Accordion>
    <Accordion title="单个机器人同时支持多少人/群的聊天？" >
        严格来说，机器人若是只挂机则无上限，具体需要根据活跃度而定，例如一条消息，普通人正常发送1~2秒，1分钟则建议20-40条消息，根据消息量和开发者可接受的的回复速度去换算同时支持的人/群。
    </Accordion>
   
    
        <Accordion title="单个Token最多支持多少微信？" >
  根据采购数量，单个Token可绑定微信数量无上限。
 </Accordion>
    
        <Accordion title="只用单个Token和使用多个Token的意义？" >
  1个Token可设置一个回调接口，采购多个机器人服务，可以配置多个Token，可设置多个回调地址。
 </Accordion>
    
        <Accordion title="一个开发者账号可以生成多少个Token？"  >
   根据采购数量，例如采购10个机器人，则最多可生成10个Token（平台默认生成1个，可通过API后台系统生成多个，也可以10个微信都登录到1个Token下），平台会记录所有Token下已登录数量是否大于或等于采购数量，等于了则无法登录新的机器人。
 </Accordion>
    
        <Accordion title="不同时间采购多个季度服务，某个服务到期后，是如何下线的？" >
服务每天会随机踢下线超出的数量。
 </Accordion>
    
        <Accordion title="RPA服务，数据安全如何保障？"  >
本服务支持公有云服务SAAS/私有化部署方式，如对数据和使用场景比较敏感，建议私有化部署，常规开发建议SAAS服务即可，SAAS服务仅是路由转发，不存储用户敏感内容数据。

 </Accordion>
    
            <Accordion title="服务有SDK吗？" >
服务目前统一HTTP接口协议调用，无业务封装SDK服务，开放性与扩展性更高。

 </Accordion>
    
            <Accordion title="是否支持机器人收到的消息并进行回复？" >
支持，设置回调消息接口，就是将收到的消息推送到您设置的接口，保证接口可接收POST 且携带JSON参数的请求。

 </Accordion>
    
            <Accordion title="回调服务是否支持配置多个回调接口？"  >
支持，一个Token可设置一个回调接口，采购多个机器人服务，可配置多个Token。
 </Accordion>
